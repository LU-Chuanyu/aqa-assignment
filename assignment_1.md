## Q3. 各 5 份 Quant 岗位 JD 调研

下表汇总了过去 12 个月内出现在投资银行 / 对冲基金 / 做市商招聘页与 LinkedIn 上的典型 JD(经过整合归纳,而非逐字搬运)。每类 5 份 JD 对应的雇主举例:

- **Quant Developer (QD):** Citadel Securities、Jane Street、Goldman Sachs Strats、JP Morgan、Optiver
- **Quant Researcher (QR):** Two Sigma、D. E. Shaw、AQR、Man AHL、Morgan Stanley QSI
- **Quant Trader (QT):** Jump Trading、IMC、Tower Research、Citadel(Fundamental Quant)、SIG

### 3.a Programming 要求总结

| 岗位 | 必备语言 | 加分语言/技术 | 工程要求 |
|---|---|---|---|
| Quant Developer | **C++(modern: C++17/20,模板、并发、低延迟优化)** 或 **Java**;Python 几乎全员要求 | Rust、Linux 内核调优、Kernel-bypass(DPDK、Solarflare)、FPGA(HFT 公司)、kdb+/q | 多线程、无锁数据结构、内存模型;熟悉网络栈(TCP/UDP/Multicast);CI/CD、Git;TDD;LeetCode Hard 级算法 |
| Quant Researcher | **Python(NumPy/Pandas/SciPy/scikit-learn/PyTorch 或 TF)**;能读/改 C++ 加分 | R、Julia、Matlab、kdb+/q;Spark/Dask 等大数据栈;SQL | 熟悉 ML pipeline、特征工程、回测框架;能将研究代码 productionize 给 QD;Jupyter + Git 工作流 |
| Quant Trader | Python(策略原型、风控脚本); shell 脚本 | C++ 读代码能力;kdb+/q;Excel/VBA(部分 sell-side) | 能在策略上线后实时监控、调参;熟悉 OMS/EMS、FIX 协议、交易所 API |

### 3.b Mathematics 要求总结

| 岗位 | 核心要求 | 加分项 |
|---|---|---|
| Quant Developer | 离散数学、算法复杂度、数值方法(舍入误差、稳定性) | 线性代数、概率论;懂金融数学有助沟通,但不强制 |
| Quant Researcher | **概率论与数理统计**(测度论级别加分)、**线性代数**、**最优化**(凸优化、随机优化)、**时间序列**(ARIMA/GARCH/状态空间)、**机器学习**(监督/无监督/强化学习) | 随机过程(布朗运动、Ito 引理)、信息论、贝叶斯统计、因果推断 |
| Quant Trader | 概率与统计(条件概率、贝叶斯、期望与方差)、**心算与博弈题**(扑克题、骰子题、市场制作题) | 随机过程、博弈论;期权希腊字母与隐含波动率直觉 |

### 3.c Financial Knowledge 要求总结

| 岗位 | 必备产品/概念 | 加分项 |
|---|---|---|
| Quant Developer | 至少了解一种资产类(equities / futures / options / FX / rates);订单簿、撮合机制、市场微结构基本概念 | 期权定价(BS/Heston)、固收(yield curve、duration、convexity)、清算结算流程 |
| Quant Researcher | **股票因子模型(Fama-French、Barra)**、**期权定价与希腊字母**、**统计套利与配对交易**、**风险模型(VaR、ES、跟踪误差)**、**回测陷阱**(look-ahead、survivorship、overfitting) | 利率衍生品、信用衍生品、ABS、加密资产、另类数据 |
| Quant Trader | 期权希腊字母(Δ、Γ、Vega、Θ)、波动率曲面、套利策略(基差、跨期、跨品种)、做市原理(inventory risk、adverse selection) | 公司行动(分红、拆股)、宏观经济日历对市场影响、做市商激励机制(maker-taker fee) |

- **想做 Quant Developer:** CS 主修 + 选修 Linear Algebra / Probability + 1 门 Derivatives + 1 门 Market Microstructure。

- **共同基础:** Python、C++、Probability、Linear Algebra、Derivatives——这五门是"无论哪条线都不能缺"的核心。

## Q5. 学习过程中的疑问 / Open Questions

下面整理在完成 Q1–Q4 调研时遇到、值得进一步与讲师 / 业界确认的问题:

1. **业务边界模糊:**
   - Multi-strategy(Millennium / Citadel)既做股票 Long/Short、统计套利,也做固收 RV 与商品宏观,应该归类成 Relative Value 还是 "Multi-strategy"? 这两者业界口径是否有统一定义?
   - Citadel 的对冲基金部门和 Citadel Securities(做市商)在投研、技术栈、风控上的真实关系是什么?是否真的"完全独立"?

2. **数据可得性:**
   - 大多数对冲基金 AUM 数据只能在 Form ADV(SEC)中找到;非美国基金(英国 FCA、香港 SFC 注册)信息透明度差,如何系统性查找?
   - "Number of employees" 在 LinkedIn 上能搜索到的 profile 通常少于真实在职人数,如何修正这种偏差?

3. **岗位定义差异:**
   - 同样叫 "Quant Researcher",在 RenTech、Two Sigma 与 AQR 的工作内容差距很大(高频信号 vs. 风险溢价 vs. 因子投资),JD 看不出来。是否有更细的细分(HFT Quant Researcher / Mid-Freq QR / Factor QR)?
   - Sell-side 的 "Strats"(Goldman/JPM)和 buy-side 的 Quant 在工作内容上有哪些本质区别?职业发展路径差异如何?

4. **技术栈选择:**
   - HFT 公司(Jane Street、Jump、HRT)使用的语言差异很大(OCaml / C++ / Rust)。在校生应该优先深入哪一门?
   - kdb+/q 在 sell-side 与 macro fund 仍是事实标准,但学校几乎不教,业界是否有标准学习资源?

5. **职业路径:**
   - 量化交易员(QT)和量化研究员(QR)在 5–10 年后的晋升路径差异?是否 QT 更容易在 30 岁后转 PM,而 QR 更容易在 multi-PM 平台被砍?
   - 对于非 STEM 博士背景的学生,是否还有进入 RenTech / D.E. Shaw 这类"博士工厂"的现实路径?
   - 跨境(美→中/港,中/港→美)从业的执照与签证门槛分别是什么?

6. **行业趋势:**
   - 大模型 / Generative AI 对传统统计套利与 NLP 因子有多大冲击?对求职者技能要求会怎样改变?
   - 加密资产是否已经成为量化基金主流配置?校招路径与传统资产是否一样?
